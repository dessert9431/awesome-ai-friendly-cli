## 🚀 What this is

awesome-ai-friendly-cli is a curated set of flags and config tips that helps command line tools stay quiet for AI agents.

It is made for people who use AI coding tools and want cleaner command output. Less noise means the agent can read results faster and use fewer tokens.

Use it to:

- hide extra prompts and progress bars
- keep output short and machine-readable
- reduce repeated text in terminal logs
- make CLI tools easier for AI agents to parse
- cut wasted tokens during agent runs

## 💻 Who this is for

This project is for you if you:

- use Windows

- want less text in terminal output
- want commands that are easier to read and automate
- want a simple list of useful flags and config patterns

You do not need to know how to code to use this repository. You only need to open the page, read the examples, and copy the parts you need.

## 🧭 How it works

Most command line tools print more than you need. They may show:

- progress bars
- color output
- help text
- prompts
- warnings you already know
- repeated status lines

This repository collects ways to turn that off or reduce it.

A quiet command is easier for an AI agent to use because:

- the output is shorter
- the result is easier to parse
- there is less chance of confusion
- fewer tokens go to extra text

## 🧪 Troubleshooting

If a command stops working after you add a quiet flag:

1. remove the new flag
2. run the command again
3. check if the tool needs a different option
4. look for a JSON or no-color option instead
5. test the command with a single change at a time

If output is still noisy, look for:

- environment variables
- config files
- default terminal colors
- wrapper scripts
- shell aliases

Some tools hide settings in config files, not in the command itself.

## 🔒 Safety and control

Quiet mode is useful, but you still want the command to show real errors.

Keep these points in mind:

- do not hide errors you need to see
- use silent mode only when the tool still reports failures well
- test commands before putting them into an agent flow
- keep a normal version of each command for checks
